Data Types in JavaScript

JavaScript provides different data types to hold different types of values. There are two main categories of data types in JavaScript:
  • Primitive Data Types
  • Non-Primitive Data Types (Objects)

Primitive Data Types:
  • String: Represents textual data. Example: "Hello, World!"
  • Number: Represents both integer and floating-point numbers. Example: 42, 3.14
  • Boolean: Represents logical values, either true or false. Example: true, false
  • Null: Represents the intentional absence of any object value. Example: null
  • Undefined: Represents a variable that has been declared but not assigned a value. Example: undefined
  • Symbol: Represents a unique identifier. Example: Symbol('description')
  • BigInt: Represents integers with arbitrary precision. Example: 9007199254740991n

Non-Primitive Data Types (Objects):
  • Object: A collection of properties, where each property is a key-value pair. Example: {name: "John", age: 30}
  • Array: A special type of object used to store ordered collections of values. Example: [1, 2, 3, 4, 5]
  • Function: A block of code designed to perform a particular task. Example: function greet() { console.log("Hello!"); }
  • Date: Represents dates and times. Example: new Date()
  • RegExp: Represents regular expressions for pattern matching in strings. Example: /ab+c/

Note:

  • Primitive data types are immutable, meaning their values cannot be changed once created.
  • Non-primitive data types (objects) are mutable, meaning their properties can be changed.
  • JavaScript is a dynamically typed language, so you don't need to specify the data type when declaring a variable. The type is determined at runtime based on the assigned value.
Example:
  • let name = "Alice"; // String
  • let age = 30; // Number
  • let isStudent = false; // Boolean
  • let address = null; // Null
  • let phoneNumber; // Undefined
  • let uniqueId = Symbol('id'); // Symbol
  • let bigNumber = 9007199254740991n; // BigInt
  • let person = {name: "Bob", age: 25}; // Object
  • let numbers = [1, 2, 3, 4, 5]; // Array
  • function greet() { console.log("Hello!"); } // Function
  • let currentDate = new Date(); // Date
  • let pattern = /ab+c/; // RegExp

Note:-

  • Note 1: representation of string data type is done by double quotes("") or single quotes('') or backticks(``)
  • Note 2: double quotes("") and single quotes('') are used for string representation but backticks(``) are used for template literals
  • Note 3: both double quotes("") and single quotes('') are used to represent string but we cannot use them interchangeably Example: let greeting1 = "Hello, World!"; // Using double quotes let greeting2 = 'Hello, World!'; // Using single quotes console.log(greeting1); // Output: Hello, World! console.log(greeting2); // Output: Hello, World!
  • Note 4: double quotes("") is an advaced version of single quotes('') because it can hold single quotes('') within it without any escape character Example: let message1 = "It's a beautiful day!"; // Using double quotes to hold single quote let message2 = 'It\'s a beautiful day!'; // Using single quotes with escape character console.log(message1); // Output: It's a beautiful day! console.log(message2); // Output: It's a beautiful day!
  • Note 5: template literals are used to embed expressions and variables within a string using ${} Example: let name = "Aadi"; let greeting = `Hello, ${name}!`; // Using backticks for template literal console.log(greeting); // Output: Hello, Aadi!
  • Note 6: template literals can also span multiple lines without the need for escape characters Example: let multiLineString = `This is a string that spans multiple lines.`; console.log(multiLineString); // Output: // This is a string // that spans multiple // lines.
  • Note 7: while using template literals, if we want to use backticks(``) within the string, we need to escape them using a backslash(\) Example: let quote = `He said, \`JavaScript is awesome!\``; // Escaping backticks console.log(quote); // Output: He said, `JavaScript is awesome!`
  • Note 8: in template literals, we can also use expressions within the ${} syntax Example: let a = 5; let b = 10; let sumMessage = `The sum of ${a} and ${b} is ${a + b}.`; console.log(sumMessage); // Output: The sum of 5 and 10 is 15.
  • Note 9: template literals are particularly useful for creating complex strings that include variables and expressions, making the code more readable and maintainable. Example: let user = { name: "Aadi", age: 25 }; let userInfo = `User Info: Name: ${user.name} Age: ${user.age}`; console.log(userInfo); // Output: // User Info: // Name: Aadi // Age: 25
