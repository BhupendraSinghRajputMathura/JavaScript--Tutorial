Event Handlers using DOM Property

Definition

An Event Handler is a JavaScript function that runs when a specific event occurs on an HTML element.
When using DOM properties, event handlers are assigned directly to elements using properties such as onclick, onmouseover, etc.

Syntax


// Syntax:
element.event = functionName;

// Example:
document.getElementById("btn").onclick = myFunction;

function myFunction() {
  alert("Button clicked!");
}

Types of Event Handlers (Using DOM Property)

There are various types of event handlers in JavaScript that can be assigned using DOM properties. Here are the most commonly used ones:

1. onclick

Definition: The onclick event fires when an element is clicked.

Syntax: element.onclick = functionName;


<button id="btn1">Click Me</button>

<script>
  let btn = document.getElementById("btn1");
  btn.onclick = function() {
    alert("Button Clicked!");
  };
</script>

2. ondblclick

Definition: The ondblclick event fires when an element is double-clicked.

Syntax: element.ondblclick = functionName;


<button id="btn2">Double Click Me</button>

<script>
  let btn2 = document.getElementById("btn2");
  btn2.ondblclick = function() {
    alert("Button Double Clicked!");
  };
</script>

3. onmouseover

Definition: The onmouseover event triggers when the mouse pointer moves over an element.

Syntax: element.onmouseover = functionName;


<div id="box1" style="width:150px;height:100px;background-color:lightblue;text-align:center;">
  Hover Me
</div>

<script>
  let box = document.getElementById("box1");
  box.onmouseover = function() {
    box.style.backgroundColor = "orange";
    box.innerText = "Mouse Over!";
  };
</script>

4. onmouseout

Definition: The onmouseout event occurs when the mouse pointer leaves an element.

Syntax: element.onmouseout = functionName;


<div id="box2" style="width:150px;height:100px;background-color:lightgreen;text-align:center;">
  Move Out
</div>

<script>
  let box2 = document.getElementById("box2");
  box2.onmouseout = function() {
    box2.style.backgroundColor = "lightgreen";
    box2.innerText = "Mouse Out!";
  };
</script>

5. onkeydown

Definition: The onkeydown event occurs when a key is pressed down.

Syntax: element.onkeydown = functionName;


<input type="text" id="textInput" placeholder="Type something...">

<script>
  let input = document.getElementById("textInput");
  input.onkeydown = function() {
    console.log("A key is pressed down!");
  };
</script>

6. onkeyup

Definition: The onkeyup event fires when the user releases a key on the keyboard.

Syntax: element.onkeyup = functionName;


<input type="text" id="input2" placeholder="Release a key...">

<script>
  let input2 = document.getElementById("input2");
  input2.onkeyup = function() {
    console.log("Key released!");
  };
</script>

7. onchange

Definition: The onchange event occurs when the value of an input element changes and the focus is lost.

Syntax: element.onchange = functionName;


<input type="text" id="nameInput" placeholder="Enter name">

<script>
  let nameInput = document.getElementById("nameInput");
  nameInput.onchange = function() {
    alert("Value Changed: " + nameInput.value);
  };
</script>

8. onload

Definition: The onload event executes a function after the page or image has fully loaded.

Syntax: window.onload = functionName;


<script>
  window.onload = function() {
    alert("Page Loaded Successfully!");
  };
</script>

Advantages of Using DOM Property Event Handlers

  • Simple and direct way to attach event listeners.
  • Easy to understand and implement.
  • Great for small scripts and single event handling per element.

Limitations

  • You can only assign one handler per event property (the previous one gets overwritten).
  • For multiple events on the same element, addEventListener() is preferred.
