NodeList()

A NodeList is a collection of nodes, which can be elements, text nodes, or other types of nodes in the DOM. It is similar to an array but does not have all the methods and properties of a standard JavaScript array. NodeLists are often returned by methods that select multiple elements, such as getElementsByClassName() or querySelectorAll().

NodeLists can be live or static. A live NodeList automatically updates when the underlying document changes, while a static NodeList does not change after it is created. For example, getElementsByClassName() returns a live NodeList, while querySelectorAll() returns a static NodeList.

To work with a NodeList, you can use a for loop or the forEach() method to iterate over its items. You can also convert a NodeList to an array using methods like Array.from() or the spread operator ([...nodeList]) to take advantage of array methods.

Overall, NodeLists are a useful way to work with collections of nodes in the DOM, allowing developers to manipulate multiple elements at once.

HTMLCollection()

An HTMLCollection is a collection of HTML elements in the DOM. It is similar to a NodeList but specifically contains only element nodes. HTMLCollections are often returned by methods that select multiple elements, such as getElementsByTagName() or getElementsByClassName().

HTMLCollections are live, meaning they automatically update when the underlying document changes. For example, if you add or remove elements from the document that match the criteria of the HTMLCollection, the collection will reflect those changes in real-time.

To work with an HTMLCollection, you can use a for loop to iterate over its items. However, HTMLCollections do not have the forEach() method like NodeLists. You can also convert an HTMLCollection to an array using methods like Array.from() or the spread operator ([...htmlCollection]) to take advantage of array methods.

Overall, HTMLCollections are a useful way to work with collections of HTML elements in the DOM, allowing developers to manipulate multiple elements at once.

Difference between NodeList and HTMLCollection

NodeList HTMLCollection
A NodeList is a collection of nodes, which can include element nodes, text nodes, and other types of nodes. An HTMLCollection is a collection of HTML elements, specifically element nodes.
NodeLists can be live or static. A live NodeList updates automatically when the underlying document changes, while a static NodeList does not change after it is created. HTMLCollections are always live, meaning they automatically update when the underlying document changes.
NodeLists can be created using methods like getElementsByClassName() (live) and querySelectorAll() (static). HTMLCollections can be created using methods like getElementsByTagName() and getElementsByClassName().
NodeLists have the forEach() method, allowing for easy iteration over their items. HTMLCollections do not have the forEach() method, so you need to use a for loop or convert them to an array to iterate over their items.
NodeLists can contain different types of nodes, including text nodes and comment nodes. HTMLCollections only contain element nodes.

Difference between innerText, textContent, and innerHTML

Property Description
innerText Gets or sets the text content of an element, including only the visible text. It does not include hidden text or HTML tags.
textContent Gets or sets the text content of an element, including all text, regardless of visibility. It includes hidden text but not HTML tags.
innerHTML Gets or sets the HTML content of an element, including all HTML tags and their content. It allows for the manipulation of the element's structure.
