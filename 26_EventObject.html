Event Object in JavaScript

Definition

The Event Object in JavaScript represents an event that takes place in the DOM (Document Object Model). Whenever an event occurs, such as a user clicking a button, pressing a key, or moving the mouse, the browser creates an event object that contains information about that event.

Syntax


element.addEventListener("eventType", function(event) {
  // Access event object properties
  console.log(event.type);
  console.log(event.target);
});

Here, the event parameter is the Event Object automatically passed to the event handler function.

Common Properties of Event Object

  • type – Returns the type of event (e.g., "click", "keydown").
  • target – Returns the element that triggered the event.
  • currentTarget – Returns the element the event listener is attached to.
  • clientX / clientY – Returns mouse coordinates (for mouse events).
  • key – Returns which key was pressed (for keyboard events).
  • preventDefault() – Cancels the default behavior of the event.
  • stopPropagation() – Prevents the event from bubbling up the DOM.

Types of Events

1. Mouse Events

These events are triggered by mouse actions such as clicking, double-clicking, hovering, or moving the mouse.

Examples:


// HTML
<button id="btn">Click Me</button>

// JavaScript
document.getElementById("btn").addEventListener("click", function(event) {
  alert("Button clicked!");
  console.log("Event Type:", event.type);
  console.log("Triggered By:", event.target.tagName);
});
  • click – When a user clicks an element.
  • dblclick – When a user double-clicks an element.
  • mouseover – When the mouse pointer enters an element.
  • mouseout – When the mouse pointer leaves an element.

2. Keyboard Events

These events occur when a user interacts with the keyboard.

Examples:


// HTML
<input type="text" id="inputBox" placeholder="Type something...">

// JavaScript
document.getElementById("inputBox").addEventListener("keydown", function(event) {
  console.log("Key pressed:", event.key);
});
  • keydown – Fires when a key is pressed down.
  • keyup – Fires when a key is released.
  • keypress – Fires when a key that produces a character value is pressed.

3. Form Events

These events occur when users interact with forms, such as submitting, resetting, or changing input values.

Examples:


// HTML
<form id="myForm">
  <input type="text" id="username" placeholder="Enter name">
  <button type="submit">Submit</button>
</form>

// JavaScript
document.getElementById("myForm").addEventListener("submit", function(event) {
  event.preventDefault(); // Prevents form submission
  console.log("Form submitted by:", event.target.id);
});
  • submit – When a form is submitted.
  • change – When the value of an input element changes.
  • focus – When an element gets focus.
  • blur – When an element loses focus.

4. Window Events

These events are triggered by actions performed on the browser window, such as loading or resizing.

Examples:


// JavaScript
window.addEventListener("load", function(event) {
  console.log("Page fully loaded!");
});

window.addEventListener("resize", function(event) {
  console.log("Window resized - Width:", window.innerWidth);
});
  • load – When the webpage finishes loading.
  • resize – When the browser window is resized.
  • scroll – When the user scrolls the webpage.

5. Clipboard Events

These events occur during cut, copy, or paste operations.

Example:


// HTML
<input type="text" id="copyBox" value="Copy this text">

// JavaScript
document.getElementById("copyBox").addEventListener("copy", function(event) {
  alert("You copied the text!");
});
  • copy – When the user copies content.
  • cut – When the user cuts content.
  • paste – When the user pastes content.

Conclusion

The Event Object is essential for handling user interactions in web applications. It gives detailed information about what happened, where it happened, and allows developers to control event behavior dynamically.
