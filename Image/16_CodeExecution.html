Code Execution Context In JavaScript

The code execution context is an abstract concept that holds information about the environment within which the current code is being executed. It includes details such as variable scope, the value of 'this', and the order of function calls.

Types of Execution Context

  • Global Execution Context: The default context where any JavaScript code runs initially.
  • Function Execution Context: Created whenever a function is invoked, containing its own scope and variables.
  • Eval Execution Context: Created by the eval function, allowing for dynamic code execution.

Execution Context Lifecycle

The lifecycle of an execution context can be broken down into three phases:

  1. Creation Phase: During this phase, the context is created, and the scope is established. Variables are hoisted, and function declarations are processed.
  2. Activation Phase: The context is activated, and the code begins to execute. The value of 'this' is determined, and the local variables are initialized.
  3. Termination Phase: Once the code has finished executing, the context is destroyed, and any resources are released.

GEC (Global Execution Context)

The Global Execution Context (GEC) is the default context where any JavaScript code runs initially. It is created when the JavaScript engine starts executing the code and serves as the outermost context.

Characteristics of GEC

  • Single instance: There is only one global execution context in a JavaScript environment.
  • Global scope: Variables declared in the global context are accessible from anywhere in the code.
  • Window object: In a browser environment, the global context is associated with the window object.

FEC (Function Execution Context)

The Function Execution Context (FEC) is created whenever a function is invoked. It contains its own scope, variables, and the value of 'this' specific to that function call.

Characteristics of FEC

  • Multiple instances: A new function execution context is created for each function call.
  • Local scope: Variables declared within the function are not accessible from outside.
  • Closure: Inner functions have access to the outer function's scope.

Eval Execution Context (EEC)

The Eval Execution Context (EEC) is created by the eval function, allowing for dynamic code execution. It has its own scope and can access variables from the surrounding context.

Characteristics of EEC

  • Dynamic scope: The scope is determined at runtime, allowing for more flexible code execution.
  • Access to outer variables: EEC can access variables from the surrounding context, similar to closures.
  • Security concerns: Using eval can lead to security vulnerabilities, as it allows for the execution of arbitrary code.

Execution Phases

The execution phases of a context can be divided into three main stages:

  1. Creation Phase: This is when the context is created, and the scope is established. Variables are hoisted, and function declarations are processed.
  2. Activation Phase: The context is activated, and the code begins to execute. The value of 'this' is determined, and the local variables are initialized.
  3. Termination Phase: Once the code has finished executing, the context is destroyed, and any resources are released.

Call Stack

The call stack is a data structure that keeps track of the execution contexts in a JavaScript program. It follows a Last In, First Out (LIFO) order, meaning that the most recently added context is the first one to be removed.

When a function is called, a new execution context is created and pushed onto the call stack. When the function returns, its context is popped off the stack.

Example

        
            var a = 10; // Global Execution Context (GEC)
            var b = 20; // Global Execution Context (GEC)

            function add(x,y) {
                var c = 30; // Function Execution Context (FEC)
                return x + y + c;
            }

            add(); // Function Execution Context (FEC) is created and pushed onto the call stack
