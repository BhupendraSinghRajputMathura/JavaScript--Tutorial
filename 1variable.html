Variable in JavaScript

A variable is a container for storing data values.

there are 3 ways to declare a variable in JavaScript:
  • var
  • let
  • const

note:

  • Variables declared with var or let can be reassigned.
  • Variables declared with const cannot be reassigned.
Example:
  • const panCard = "1234-5678-9012-3456";
  • let age = 25;
  • var name = "John Doe";

  1. In the above example, panCard is a constant variable that cannot be changed, age is a variable that can be reassigned, and name is a variable that can also be reassigned.
  2. Prefer using const for variables that should not change, and let for variables that may change. Avoid using var to prevent scope-related issues.

interview Question: What is the difference between var, let, and const in JavaScript? difference between var, let and const:
  • Scope:
    • var: Function-scoped. Accessible throughout the function in which it is declared.
    • let: Block-scoped. Accessible only within the block (enclosed by {}) in which it is declared.
    • const: Block-scoped. Accessible only within the block in which it is declared.
  • Reassignment:
    • var: Can be reassigned.
    • let: Can be reassigned.
    • const: Cannot be reassigned after its initial assignment.
  • Hoisting:
    • var: Hoisted to the top of its scope and initialized with undefined.
    • let: Hoisted to the top of its block but not initialized. Accessing it before declaration results in a ReferenceError (temporal dead zone).
    • const: Hoisted to the top of its block but not initialized. Accessing it before declaration results in a ReferenceError (temporal dead zone).
  • Use Cases:
    • var: Generally avoided in modern JavaScript due to its function scope and hoisting behavior, which can lead to unexpected results.
    • let: Used for variables that may need to be reassigned, such as loop counters or values that change over time.
    • const: Used for variables that should remain constant throughout the program, such as configuration values or references to objects/arrays that should not be reassigned.
Example:
  • var x = 10; // function-scoped, can be reassigned
  • let y = 20; // block-scoped, can be reassigned
  • const z = 30; // block-scoped, cannot be reassigned
