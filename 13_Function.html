JavaScript Functions - Complete Notes

What is a Function?

A function is a block of code designed to perform a specific task. It runs when it is called or invoked.

Defining a Function

function greet() {
  console.log("Hello, World!");
}

Calling a Function

greet(); // Output: Hello, World!

Function with Parameters

function add(a, b) {
  return a + b;
}

console.log(add(5, 10)); // 15

Arrow Functions

const multiply = (x, y) => x * y;
console.log(multiply(4, 5)); // 20

Default & Rest Parameters

function multiply(a, b = 1) {
  return a * b;
}

function sum(...nums) {
  return nums.reduce((a, b) => a + b, 0);
}

this Keyword in Functions

const obj = {
  value: 42,
  regular: function() {
    console.log(this.value);
  },
  arrow: () => {
    console.log(this.value);
  }
};

obj.regular(); // 42
obj.arrow();   // undefined
Note: Arrow functions do not have their own this — they inherit it from the surrounding scope.

Function Hoisting

sayHello(); // Works due to hoisting

function sayHello() {
  console.log("Hello!");
}

Passing Objects and Arrays

const person = { name: "Alice", age: 25 };
function greet({ name, age }) {
  console.log(`Hi, I'm ${name} and I'm ${age}.`);
}

const numbers = [1, 2, 3];
function sumArray(arr) {
  return arr.reduce((a, b) => a + b, 0);
}

Higher-Order & Callback Functions

function higherOrder(fn) {
  fn();
}

function callback() {
  console.log("Callback executed");
}

higherOrder(callback);
Real-World Use: Higher-order functions like map(), filter(), and reduce() rely on callbacks to process data efficiently.

Key Takeaway

  • Use function for reusable tasks.
  • Use => for concise logic or callbacks.
  • Understand how this behaves differently.
